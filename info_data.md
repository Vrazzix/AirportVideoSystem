# Документация проекта — Application for models

## Датасеты и данные

### augmented/
- Датасет (колодки + колеса) аугментированный, не сплитованный (830 изображений)
- Используется для тренировки базовой модели YOLO


### boots_dataset/
- Сырой датасет колодок (boots = chocks)
- Содержит оригинальные изображения до аугментации и сплита

### downloaded_images/
- Скачанные изображения с поиска по запросам (колеса, колодки, люди)
- Используется как материал для пополнения датасета

### train/
- Финальный тренировочный сет для YOLOv8
- Структура: images/, labels/ (YOLO txt-формат)
- ~83 подготовленных изображения

### hard_negatives/
- Изображения, на которых модель ошибается (false positives)
- Используются для переобучения и улучшения точности

### auto_labeled/
- Автоматически размеченные изображения (может быть, по средней уверенности)
- Кандидаты для ручной верификации

### data.yaml
- Конфиг YOLOv8 с путями к train/val/test и классами
- Используется при запуске тренировки (например, `yolo detect train data=data.yaml`)

## Основной код

### app (4).py
- **Главное приложение** — Streamlit Web UI
- Функции:
  - Загрузка видеофайла
  - Запуск детекции (YOLO: колёса, колодки, люди)
  - Поза человека (MPII 16 keypoints)
  - IoU-based трекинг объектов (SimpleTracker)
  - **Semantic SLAM** — верификация объектов по 3D-структуре
  - Вывод обработанного видео + 3D карта bird's-eye
- Параметры в sidebar: пороги детекции, фильтрация FP, трекинг, SLAM

### main.py
- Точка входа для локального запуска
- Может содержать простой test script или вспомогательные функции

### train_finetune.py
- **Скрипт для дообучения/фин-тюнинга** YOLOv8 модели
- Входные данные: data.yaml + датасет в train/
- Выходные данные: trained weights в models/

## Утилиты для подготовки данных

### scrape_images.py
- Скрипт скачивания изображений по поисковым запросам
- Используется для сбора raw-датасета

### augmentation.py
- Аугментирует датасет (вращение, масштабирование, шум, etc.)
- Входные данные: оригинальные изображения
- Выходные данные: увеличенный датасет в augmented/

### auto_label (2).py
- Автоматическая разметка изображений (детекция + создание YOLO txt-файлов)
- Использует предтренированную модель для предварительной разметки

### hard_negatives.py
- Анализирует ошибки модели (false positives)
- Выделяет сложные примеры для переобучения
- Выходные данные: hard_negatives/ с проблемными изображениями

### split_dataset.py
- Разделяет датасет на train/val/test в нужном формате для YOLO
- Структурирует папки и labels

## Модели

### models/
- **bestBoots_v2.pt** — YOLOv8 модель для детекции колёс и колодок (кастомная тренировка)
- **BestPose.pt** — YOLOv8-Pose модель для детекции позы человека (MPII 16 keypoints)
- **yolov8n.pt** — базовая YOLOv8 Nano для детекции людей (скачивается автоматически)
