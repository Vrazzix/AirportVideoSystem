# AirportVideoSystem

AirportVideoSystem - веб-приложение для анализа видео аэропортового обслуживания. Система помогает находить объекты и события на видео: людей, колодки у колес, позу сотрудника, состояние двери и этапы обслуживания воздушного судна.

## Что нужно для запуска

Перед запуском установите:

- Python 3.10 или новее.
- Git, если проект скачивается из репозитория.
- Веса моделей YOLO/OpenVINO, если нужно выполнять реальное распознавание на видео.

## Установка

Скачайте проект и перейдите в папку приложения:

```bash
git clone https://github.com/Vrazzix/AirportVideoSystem.git
cd AirportVideoSystem/AirportVideoSystem
```

Создайте виртуальное окружение:

```bash
python -m venv .venv
```

Активируйте окружение в Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

Если используется Linux или macOS:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Подготовка моделей

Веса моделей не хранятся в репозитории, потому что это крупные бинарные файлы. Для работы распознавания создайте папку `models` внутри приложения:

```bash
mkdir models
```

Поместите туда модели со следующими именами:

- `models/BestBoots_v2.pt` - модель для поиска колодок.
- `models/person.pt` - модель для поиска людей.
- `models/BestPose.pt` - модель оценки позы.
- `models/door.pt` - модель двери, если она используется.

Модели также можно загрузить через веб-интерфейс приложения. Поддерживаются форматы `.pt`, `.onnx`, `.engine`, `.tflite`, `.pb` и OpenVINO-модели `.xml` + `.bin`.

## Запуск приложения

Запустите сервер:

```bash
python web_app.py
```

После запуска откройте в браузере:

```text
http://localhost:5000
```

## Как пользоваться

1. Откройте веб-интерфейс.
2. Загрузите видео для анализа.
3. Проверьте или укажите пути к моделям в настройках.
4. Запустите обработку.
5. Посмотрите результаты детекции, трекинга, событий и визуализации.

## Структура проекта

- `AirportVideoSystem/` - основное Flask-приложение.
- `AirportVideoSystem/web_app.py` - точка входа для запуска сервера.
- `AirportVideoSystem/requirements.txt` - список зависимостей.
- `AirportVideoSystem/modules/` - обработка видео, детекция, события, трекинг, SLAM и визуализация.
- `AirportVideoSystem/templates/` - HTML-интерфейс.
- `AirportVideoSystem/sources/` - вспомогательные модули.

## Что не входит в репозиторий

В GitHub не загружаются датасеты, видео, архивы, результаты экспериментов, веса моделей и материалы дипломной работы. Эти файлы должны храниться локально отдельно от кода приложения.
